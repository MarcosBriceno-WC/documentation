The current page reads like a feature inventory rather than a task guide. It lists options without orienting the reader, uses sections that are too short to act on, and relies on lists where numbered steps would serve better. Instructions also lack the context needed to execute them. Telling users to "Click Connect MCP at the bottom of the sidebar" without explaining where that sidebar lives, or providing configuration file snippets without specifying which file to edit or where to find it, leaves users stuck. Someone unfamiliar with MCPs would need to leave the page entirely just to find a file path, which is exactly where onboarding drops off.

The rewrite reorganizes the page around the three connection methods as distinct, sequential flows, each with numbered steps, prerequisites, and a verification moment so users know when they're done. Client-specific instructions were expanded to cover Claude Desktop (with config paths for macOS and Windows), Claude Code, VS Code, and Windsurf, since the original assumed users already knew which method they wanted.

If this were my account, I'd apply the same treatment to Getting Started, which currently reads like a glossary rather than a directed path, and give Skills and Commands the same structural overhaul. I'd also add a decision guide early in the docs to help users choose between connection methods before they land on the wrong page and have to backtrack.
